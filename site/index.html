QAN Agent

不只是"慢查询列表"

每个 SQL 指纹关联 12+ 维度的指标：总耗时、平均延迟、执行次数、扫描行数、锁等待时间、临时表、文件排序、无索引比例……从汇总到明细，一层层下钻。

  • PerfSchema / pg_stat_statements 双数据源
  • 皮秒级时间 → 秒，自动单位换算
  • 增量 delta 计算，支持 truncate 自动恢复
  • 24h 趋势图 + 优化建议自动生成
  • 搜索、排序、Schema 过滤、时间范围切换

锁链拓扑可视化 — 谁阻塞了谁，一眼看清

实时检测行锁等待，有向图渲染，DFS 死锁环自动识别

实时拓扑

不解析 INNODB STATUS，直接查 data_lock_waits

传统做法靠正则匹配 SHOW ENGINE INNODB STATUS 文本，跨版本格式不稳定。PharosDB 直接从 performance_schema.data_lock_waits 查询，用 DFS 算法在关系数据上做环检测，精确且版本无关。

  • 支持 MySQL 5.7 / 8.0+ / MariaDB 10.6+ 自动切换 SQL
  • 阻塞方(BLOCKER) / 等待方(WAITER) / 链式中间(BOTH) 三色区分
  • 死锁节点黄色高亮 + 脉冲动画
  • 自适应轮询：无锁 30s → 检测到锁 5s → 锁清除恢复 30s
  • 等待时长、锁模式、锁对象全部展示

执行计划对比 — 历史版本 Diff，优化效果可量化

自动 EXPLAIN 采集 + 结构化摘要 + 逐节点对比，PMM 不具备的能力

Plan Diff

不只是看一次 EXPLAIN，而是看它怎么变

PMM 只展示单次 EXPLAIN 结果。PharosDB 每 60s 自动对 Top-5 慢查询执行 EXPLAIN，提取结构摘要去重保存，形成历史版本时间线。当执行计划发生变化时，逐节点 Diff 精确告诉你哪个表的访问类型变了、哪个索引被用上了、filesort 是否消失。

  • normalize_plan_summary() — 递归遍历 JSON 树，丢弃 cost 浮动值，保留结构
  • 两级去重缓存 — 内存 + ClickHouse，95%+ 命中率
  • access_type rank 排序: system < const < eq_ref < ref < range < index < ALL
  • 逐节点比较 → 优化(绿) / 退化(红) / 修改(灰) 三态标签
  • 手动 EXPLAIN 入口，即时验证索引优化效果

智能告警中心 — ClickHouse 聚合触发，状态机自动流转

四类规则引擎 · Alertmanager v4 Webhook · 无需 Prometheus

Alert Engine

告警不依赖外部系统，闭环在 ClickHouse 内完成

PMM 依赖 Prometheus AlertManager，与 ClickHouse 数据源割裂。PharosDB 的告警引擎直接查询 pharos_db.metrics 表，支持 4 类规则：慢查询耗时阈值 / 无索引比例 / 查询总量 / 自定义 SQL。每个 (规则, 实例) 独立状态机，firing → resolved 自动流转。

  • slow_query_time — 统计周期内平均耗时超标的查询类型数
  • no_index_ratio — 无索引查询占比 %
  • query_count — 总查询量异常波动
  • custom_sql — 任意 ClickHouse SQL，单行单列阈值比较

Feature Demo

零依赖内置调度 — 无 Redis·无 Celery·漂移消除

threading.Timer 链式调度，时间锚点消除秒级漂移，重启自动恢复

Agentless Core

不装 Agent，不搭 Redis，一个 Python 进程搞定全部调度

传统方案需要 Celery + Redis + Beat 三件套，部署复杂、运维成本高。PharosDB 用 threading.Timer 链式自调度 替代，每实例独立线程、独立采集间隔。通过 run_start 时间锚点消除累积漂移——第 N 次采集永远在第 N 分钟开始，不差毫秒。

  • 无 Redis / Celery / Beat 依赖，pip install 即用
  • run_start 锚点消除 Threading Timer 累积漂移
  • Django 启动时自动从 DB 恢复全部 active 实例调度
  • 每实例独立间隔 (30s ~ 3600s)，互不干扰
  • 锁采集自适应调速: 空闲 30s → 活跃 5s
