核心能力

专注那些现有工具做不好的事情

📊

SQL 分析（QAN）

基于 performance_schema 或 pg_stat_statements 的慢查询聚合。SQL 指纹识别、执行频次统计、耗时分布、扫描行数分析，支持按服务和时段过滤。Top-N 慢查询一目了然。

🔐

锁链拓扑可视化

实时检测行锁等待链，渲染为有向图。谁阻塞了谁、等了多久、涉及哪张表。DFS 算法自动检测死锁环，不依赖 INNODB STATUS 文本解析。自适应 30s / 5s 轮询。

🔔

告警中心

四种告警规则：慢查询耗时、无索引比例、查询总量、自定义 SQL。ClickHouse 聚合触发，Webhook 通知兼容 Alertmanager v4。firing / resolved 状态机自动管理。

🧪

执行计划对比

自动对 top-5 慢查询执行 EXPLAIN，持久化到 ClickHouse。历史版本时间线，逐节点 diff 对照——访问类型变化、索引增减、filesort 出现，一目了然。

⚡

零依赖调度器

内置线程调度器，无需 Redis / Celery。每个实例独立采集间隔，run_start 时间锚点消除秒级漂移。瞬时重启自动从 DB 恢复所有调度状态。

🌐

中英文 + 暗色主题

全量 CSS 变量驱动亮/暗色切换。react-i18next 国际化，中英文一键切换。Vite 5 + React 18，首屏加载 < 200KB。
