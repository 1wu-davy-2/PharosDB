PharosDB 是一款专为打破"监控孤岛"而生的数据库可观测性平台。
它把慢查询、锁等待、执行计划、基础设施这些分散的信号，
融合进同一张诊断视图——不只告诉你"哪里慢"，更指引你"为什么慢"。

核心能力

专注那些现有工具做不好的事情

📊

SQL 洞察（QAN）

基于 performance_schema / pg_stat_statements 的慢查询聚合引擎。SQL 指纹识别 · 执行频次 · 耗时分布 · 扫描行数，Top-N 一目了然。

🕸

锁链拓扑可视化

实时检测行锁等待，渲染为有向图。谁阻塞谁、等了多久、涉及哪张表。DFS 算法自动识别死锁环，无需解析 INNODB STATUS 文本。

🔔

智能告警中心

四类规则：慢查询耗时 / 无索引比例 / 查询总量 / 自定义 SQL。ClickHouse 聚合触发，Webhook 兼容 Alertmanager v4，状态机自动流转。

🧪

执行计划对比

自动对 Top-5 慢查询执行 EXPLAIN 并持久化。历史版本时间线，逐节点 Diff——索引增减、filesort 出现，一眼看出优化前后的差异。

⚡

零依赖内置调度

无 Redis，无 Celery。线程级调度器内置于服务，每实例独立采集间隔，时间锚点消除秒级漂移，重启后自动从数据库恢复全部调度状态。

🌐

多语言 · 暗色主题

全量 CSS 变量驱动亮/暗模式一键切换，react-i18next 中英文国际化。Vite 5 + React 18，首屏加载 < 200 KB，零额外运行时依赖。
